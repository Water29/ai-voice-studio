# ADR-002: 使用 JSON 文件替代数据库

## 状态
✅ 已采纳（2024-06-08）

## 背景
原 plan.md 方案使用 Prisma + SQLite 存储历史记录。用户认为不需要数据库，改为文件存储。

## 决策
使用 **JSON 文件 (`data/history.json`)** 作为数据存储方案，不引入任何数据库。

## 理由

### 适用性
- 数据量极小：最多 100 条记录，每条约 500 字节，总计 < 50KB
- 单用户/本地使用：无需并发控制、事务、索引优化
- 数据结构简单：无需关联查询

### 优势
1. **零配置**：不需要安装数据库、创建表、运行迁移
2. **可调试**：直接打开 JSON 文件即可查看数据
3. **可备份**：复制文件即可完成数据备份
4. **部署简单**：Vercel 等 Serverless 平台可直接运行

### 劣势（可接受）
- 不支持复杂查询：但对 100 条记录的全量过滤已足够
- 无并发控制：但本项目为单用户使用场景
- 文件损坏风险：通过 try-catch 和自动修复机制缓解

## 后果
- 去除了 Prisma、SQLite 等依赖
- 历史数据存储逻辑集中在 `src/lib/storage.ts`
- 历史记录上限 100 条，超限自动删除最旧记录
- 删除记录时自动清理关联的音频文件
